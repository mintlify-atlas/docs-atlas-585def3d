title: "GMP Prometheus: Prometheus with native Google Cloud export"
description: "GMP Prometheus forks upstream Prometheus to add a built-in GCM gRPC export pipeline, Remote Write 2.x proxy, HA lease, and Kubernetes secrets integration."
---

GMP Prometheus extends the upstream Prometheus monitoring system with native, in-process export of metrics to [Google Cloud Managed Service for Prometheus](https://cloud.google.com/stackdriver/docs/managed-prometheus). It runs as a drop-in replacement for the standard Prometheus server — keeping full PromQL, alerting, and TSDB compatibility — while adding a high-throughput gRPC export pipeline to Cloud Monitoring (GCM), a Remote Write 2.x proxy (`prw2gcm`), Kubernetes-native secret management, and optional HA lease-based leader election.

## What GMP Prometheus adds to upstream

<CardGroup cols={2}>
  <Card title="In-process GCM export" icon="cloud-arrow-up">
    Metrics are exported directly from the Prometheus WAL to Cloud Monitoring via gRPC — no extra remote write adapter needed.
  </Card>
  <Card title="Remote Write 2.x proxy" icon="network-wired">
    The `prw2gcm` binary accepts Remote Write 2.x from any Prometheus-compatible sender and forwards to GCM.
  </Card>
  <Card title="Hot-reloadable config" icon="rotate">
    Export credentials, metric filters, and query project IDs reload at runtime via SIGHUP without restarting.
  </Card>
  <Card title="Kubernetes secrets" icon="key">
    A Kubernetes secret provider gives each scrape target its own credentials via a native K8s watch loop.
  </Card>
</CardGroup>

## Get started

<Steps>
  <Step title="Set external labels">
    Add `project_id` and `location` to `global.external_labels` in your `prometheus.yml`. GMP requires these to route metrics correctly.
  </Step>
  <Step title="Configure GCM credentials">
    Pass `--export.credentials-file` pointing to a Google service account JSON key, or use Workload Identity on GKE.
  </Step>
  <Step title="Run the server">
    Start the GMP Prometheus binary with your project and location flags set.
  </Step>
  <Step title="Verify export">
    Check `gcm_export_samples_exported_total` on the `/metrics` endpoint to confirm samples are flowing to Cloud Monitoring.
  </Step>
</Steps>

<Note>
  This repository is a fork of [github.com/prometheus/prometheus](https://github.com/prometheus/prometheus). The `main` branch tracks upstream Prometheus. GMP-specific additions live under the `google/` directory and in select GMP release branches.
</Note>
